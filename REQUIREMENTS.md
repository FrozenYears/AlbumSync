# AlbumSync 需求文档

> 局域网内将 Android 手机相册照片备份到 Windows 11 PC 的桌面软件
>
> 版本：v0.1（需求冻结）  ·  日期：2026-06-07

---

## 1. 项目目标

- **核心目标**：用户在家庭/办公室局域网内，把 Android 手机相册（DCIM、Pictures、社交软件接收图片等"全相册"）增量、可靠地备份到 PC，并能在手机失而不复时从 PC 恢复。
- **非目标**：
  - 不做云端同步、跨网传输。
  - 不做相册浏览/检索/AI 整理。
  - 不做安卓客户端开发——复用现成 FTP 服务器 App。
  - 不做 iOS 设备支持。

## 2. 架构总览

```
┌──────────────────────┐        FTP (LAN)         ┌────────────────────────┐
│  Android 手机         │ ◀──────────────────────▶ │  Windows 11 PC          │
│  Primitive FTPd       │   控制端口 1024（默认）    │  AlbumSync.exe          │
│  暴露 /sdcard 等目录   │                         │  Tauri（Rust + React）  │
└──────────────────────┘                          └────────────────────────┘
```

- **手机端**：用户在 F-Droid / 应用商店安装 [Primitive FTPd](https://github.com/wolpi/prim-ftpd) 这一个 App，作为只读 FTP 服务器，暴露内部存储。本项目**不开发任何 Android 代码**。
- **PC 端**：单一可执行文件（带 GUI），主动扫描手机 FTP 上的相册目录，与本地清单做 diff，下载新增/变更文件、软删除消失的文件。

## 3. 功能需求

### 3.1 备份范围

- 备份 Android `/sdcard` 下与"图片/视频"相关的全部子目录，覆盖：
  - 相机拍摄：`DCIM/Camera/`
  - 截图：`Pictures/Screenshots/`
  - 社交软件接收图片：`Pictures/WeiXin/`、`Tencent/MicroMsg/WeiXin/`、`Tencent/QQ_Images/` 等
  - 其它图片/视频文件（按 MIME / 扩展名识别）
- 扫描白名单与黑名单**通过设置项可调**，默认提供一组合理预设。
- 备份内容包含：照片（jpg/png/heic/webp/raw 等）、视频（mp4/mov/3gp/mkv 等）。

### 3.2 同步语义：镜像 + 30 天软删除

- 远端有、本地无 → **下载**到本地原始相对路径。
- 远端与本地相对路径相同、`(size, mtime)` 不一致 → **重新下载**覆盖。
- 远端无、本地有 → **移动**到本地 `.trash/{相对路径}/{deleted_at}/`，并写入 SQLite 中的 `trash` 表，记录到期时间 = 删除时间 + 30 天。
- 后台任务每天清理一次过期的 trash 条目。
- 用户可在 UI"回收站"页**手动恢复**未到期的删除项。

### 3.3 触发方式

- 仅"**手动点击**同步按钮"。
- UI 上有"立即同步"按钮；托盘菜单也有"立即同步"。
- 同步运行期间可点"停止"中断（已下载完整的文件保留）。

### 3.4 设备连接

- 在 GUI 中手动填写：手机 IP、FTP 端口（默认 1024）、用户名、密码。
- 后台每 5 秒一次 TCP 探测控制端口，UI 上显示在线/离线指示灯。
- 离线状态点击"立即同步"会弹提示要求先连通。
- 密码存储：**Windows Credential Manager**（通过 `keyring` crate），不在配置文件里以明文落盘。

### 3.5 文件组织（PC 端）

- 保留手机原始目录层级：例如
  - `<备份根>/DCIM/Camera/IMG_20250603_142055.jpg`
  - `<备份根>/Pictures/Screenshots/Screenshot_2025-06-03.png`
- 备份根目录由用户在首次启动引导中选择，默认建议 `<用户目录>\Pictures\PhoneBackup`。
- 软删除目录：`<备份根>/.trash/{原相对路径}__{YYYYMMDD-HHMMSS}/原文件名`

### 3.6 首次全量同步

- 第一次同步前，先做一次"清单预览"：远端文件数、总大小、本地磁盘可用空间。
- 弹窗提示：
  - 估算耗时与占用空间
  - **镜像同步语义**（强提示：手机删了 PC 也会进回收站，30 天后真删）
- 用户确认后开始。

### 3.7 增量判定

- 算法：相对路径 + 文件大小（字节）+ FTP `MDTM`/`MLSD` 给出的修改时间。
- 不算 hash（在局域网 FTP 上代价太高，且原始文件极少在不改名的情况下被修改）。

### 3.8 进度与可观测性

- 同步页实时展示：
  - 总文件数 / 已完成 / 已跳过 / 失败
  - 当前下载文件名 + 速度（MB/s）
  - 估算剩余时间
- 历史页：最近 N 次同步运行记录（开始时间、耗时、新增/更新/删除条数、错误数）。
- 回收站页：可按文件名搜索、批量恢复、立即清空。

### 3.9 启停 / 托盘 / 自启

- **关闭窗口（X 按钮）**：最小化到系统托盘，进程继续运行。
- **真正退出**：通过托盘菜单"退出"。
- **开机自启**：设置页里提供开关，默认**关**。

## 4. 非功能需求

### 4.1 性能

- 单台设备：4 路并发下载。
- 支持断点续传（FTP `REST` 命令）。
- 大文件（>200MB）边下边显示百分比。
- 整体目标：5000 张照片首次同步 ≤ 30 分钟（视手机/Wi-Fi 而定）。

### 4.2 可靠性

- 任何一次同步失败应可重试，不留半成品文件（先下到 `*.part`，完成后原子重命名）。
- 进程崩溃 / 关机后下次启动应能从中断点继续：增量同步本身天然幂等。
- 数据库写入用事务包裹；保留 WAL 模式。

### 4.3 安全

- FTP 协议本身明文，**仅在局域网内可用**——README 中明示。
- 密码通过 OS keyring 存储。
- PC 与手机通信不引入额外第三方云。

### 4.4 体积与依赖

- 安装包 ≤ 20MB（Tauri 优势）。
- 运行时无需用户额外装 .NET、JRE、Python 等。

### 4.5 平台

- 仅 Windows 11 x64（暂不做 Linux/macOS 适配，但代码尽量保持可移植）。

## 5. 用户操作流（Happy Path）

1. 安装 AlbumSync.exe → 启动 → 首次引导。
2. 引导第 1 步：选择 PC 备份根目录。
3. 引导第 2 步：教程图，告知"手机端装 Primitive FTPd → 启动 → 记录 IP 和端口"。
4. 引导第 3 步：填手机 IP、端口、用户名、密码 → 点"测试连接"→ 通过。
5. 主界面：点"立即同步"。
6. 弹窗：本次将下载 5320 张 / 12.4 GB；磁盘剩余 512 GB；确认。
7. 进度条跑完，提示"完成，新增 5320 / 更新 0 / 删除 0 / 失败 0"。
8. 之后任何时候点托盘"立即同步"，做增量。

## 6. 风险与约束

| 风险 | 影响 | 缓解 |
|---|---|---|
| 手机 IP 因 DHCP 续约变化 | 同步失败 | 引导用户在路由器为手机绑定静态 IP；UI 显示离线时给出排查提示 |
| Primitive FTPd 后台被系统杀掉 | 同步中断 | README 说明如何加入"省电豁免"；同步前自动测连接 |
| 手机存储满或文件正在写入 | 偶发下载失败 | 单文件失败自动重试 3 次，超出后记录到错误列表，整体不中断 |
| 用户误删手机照片 | 30 天后 PC 也清掉 | 回收站 + 30 天宽限期；UI 醒目标识"距清理还剩 X 天" |
| 大量小文件下载慢 | 体验差 | 4 路并发 + FTP 连接复用 |
| 镜像同步首次跑就是几十 GB | 长时间占用网络 | 首次给出预估 + 用户可在中途暂停 |

## 7. 技术栈（已锁定）

- **PC 端框架**：Tauri 2（最新稳定版）
- **后端语言**：Rust（edition 2021）
- **前端**：React 18 + TypeScript + Vite + Tailwind CSS
- **FTP 客户端**：`suppaftp`（async-tokio 特性）
- **数据库**：SQLite（`sqlx` 或 `rusqlite`，设计阶段定）
- **密码存储**：`keyring` crate（Windows Credential Manager）
- **包管理**：`pnpm`（前端）+ `cargo`（后端）
- **打包**：`tauri build` → MSI / NSIS

## 9. 验收标准

- [ ] 在测试机上完成首次全量同步且文件计数一致。
- [ ] 第二次空跑（无新增）3 秒内提示"无变更"。
- [ ] 手机删一张照片，下次同步能正确进入 `.trash/` 且 UI 中可恢复。
- [ ] 同步中拔掉手机 Wi-Fi，进程不崩，重新连上后再次同步能续传。
- [ ] 关闭主窗口后进程仍在托盘；托盘"退出"才真正退出。
- [ ] 密码不出现在任何配置文件中（手工 grep 确认）。
- [ ] 安装包体积 ≤ 20MB。

## 10. 词汇表

- **远端 / 远程**：指 Android 手机 FTP 服务器。
- **本地**：指 PC 上的备份根目录。
- **清单**：某次扫描得到的 `(相对路径, 大小, mtime)` 列表。
- **trash / 回收站**：本项目的软删除目录，与系统回收站无关。
- **`.part` 文件**：下载未完成的临时文件，下完后原子重命名为正式文件。
